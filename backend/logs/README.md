# Backend Logs Directory

This directory contains application log files generated by the Go backend.

## Log Files

Application logs are written to this directory when the backend is running.

- **app.log**: Main application log (if configured)
- **error.log**: Error-specific logs (if configured)
- **access.log**: HTTP access logs (if configured)

## Log Rotation

For production deployments, configure log rotation to prevent disk space issues:

### Using logrotate (Linux)

Create `/etc/logrotate.d/botrix`:

```
/path/to/botrix/backend/logs/*.log {
    daily
    rotate 14
    compress
    delaycompress
    notifempty
    missingok
    create 0644 botrix botrix
}
```

### Using systemd (Linux)

If running as a systemd service, logs are managed by journald:

```bash
# View logs
journalctl -u botrix-backend -f

# View logs from last hour
journalctl -u botrix-backend --since "1 hour ago"

# Export logs
journalctl -u botrix-backend > backend.log
```

## Development Logging

In development mode, logs are also written to stdout for easier debugging.

Set environment variable:

```bash
SERVER_ENVIRONMENT=development
```

## Log Levels

Configure log level in backend/.env:

```bash
# LOG_LEVEL options: debug, info, warn, error
LOG_LEVEL=info
```

## Viewing Logs

### Real-time monitoring

```bash
# Linux/macOS
tail -f backend/logs/app.log

# Windows PowerShell
Get-Content backend/logs/app.log -Wait -Tail 50
```

### Search logs

```bash
# Linux/macOS
grep "ERROR" backend/logs/app.log

# Windows PowerShell
Select-String -Path backend/logs/app.log -Pattern "ERROR"
```

## Cleaning Old Logs

```bash
# Remove logs older than 7 days (Linux/macOS)
find backend/logs -name "*.log" -mtime +7 -delete

# Remove all logs (use with caution!)
rm backend/logs/*.log
```

## Security

- Log files may contain sensitive information
- Never commit logs to version control
- Implement proper log rotation in production
- Secure log access with appropriate file permissions
